Warum ist der Schritt so gross?

In vielen Projekten faellt der groesste Teil der Zeit nicht auf das Modell, sondern auf Datenvorbereitung. Das ist keine Nebensache, sondern der eigentliche Hebel fuer gute oder schlechte Vorhersagen.

Best Practice als Ablauf

Datensatz → Train-Test-Split → fit nur auf Train → transform auf Test
Rohdaten → Features → Modell → ehrlicher Test
Wenn du vor dem Split imputierst oder skalierst, kennt dein Modell schon Informationen aus den spaeteren Testdaten.

# Daten laden
import pandas as pd
df = pd.read_csv("penguins.csv")

# Features und Ziel trennen

categorical_features = ["species", "island", "sex"]
target_variable = "body_mass_g"

X = df[numerical_features + categorical_features]
y = df[target_variable]

# Train-Test-Split (immer zuerst!)
from sklearn.model_selection import train_test_split
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42, shuffle=True
)

Imputation: Fehlende Zutaten ergaenzen

Analogie: Wenn beim Kochen Salz fehlt, ersetzt du es nicht zufaellig, sondern mit einer plausiblen Regel.

ca. 30 Min.

Strategien auf einen Blick

Strategie Beschreibung Wann? Kueche

Data Leakage
.fit() nur auf Trainingsdaten. Auf Testdaten nur .transform().

SimpleImputer in Worten

.fit() = Der Imputer schaut auf Trainingsdaten und merkt sich den Ersatzwert.

.transform() = Der gemerkte Wert wird spaeter eingesetzt.

.fit_transform() = Beides zusammen, aber nur auf Trainingsdaten sinnvoll.

Median-Formel oeffnen
\( \mathrm{Median}(x) = \begin{cases} x_{\left(\frac{n+1}{2}\right)} & \text{wenn } n \text{ ungerade} \\ \frac{x_{\left(\frac{n}{2}\right)} + x_{\left(\frac{n}{2}+1\right)}}{2} & \text{wenn } n \text{ gerade} \end{cases} \)

Sortiere zuerst alle vorhandenen Werte. Bei ungerader Anzahl nimmst du den echten Mittelwert der Positionen. Bei gerader Anzahl mittelt man die beiden mittleren Werte.

Categorical Encoding

Analogie: Muelltrennung, Schulnoten und Groessenklassen zeigen, dass manche Kategorien keine Reihenfolge haben und andere schon.

ca. 30 Min.

OneHotEncoder oder OrdinalEncoder?

Nominal
Keine Reihenfolge. Beispiele: Tierart, Insel, Farbe, Bundesland. Meist sicher mit One-Hot-Encoding.
Ordinal
Reihenfolge zaehlt. Beispiele: Schulnote, Energieklasse, T-Shirt-Groesse. Dann ist OrdinalEncoder sinnvoll.
Wenn du nominale Daten ordinal codierst, erfindest du kuenstliche Abstaende wie "Apfel < Banane < Kirsche".

Modul 3 | Alles auf eine Waage

Feature Scaling

Analogie: Einkommen in Euro und Alter in Jahren darf das Modell nicht nur wegen der Groesse der Zahlen ungleich gewichten.

ca. 30 Min.

Skalierungsarten und Formeln

MinMaxScaler
\( x_{\mathrm{scaled}} = \frac{x - x_{\min}}{x_{\max} - x_{\min}} \)

Alle Werte werden auf den Bereich zwischen 0 und 1 gestaucht. Das ist anschaulich, aber ausreisser-empfindlich.

StandardScaler
\( z = \frac{x - \mu}{\sigma} \)

Der Mittelwert wird 0, die Standardabweichung 1. Das passt oft gut, wenn die Verteilung halbwegs normal wirkt.

RobustScaler
\( x_{\mathrm{robust}} = \frac{x - \mathrm{median}(x)}{\mathrm{IQR}} \)

Median und Interquartilsabstand reagieren viel weniger empfindlich auf extreme Ausreisser.

Log-Transformation
\( x_{\log} = \log(x + c) \)

Hilft bei schiefen Verteilungen, funktioniert aber nur fuer positive Werte.

Wann ist Scaling wichtig?

Distanzbasierte Verfahren wie KNN und SVM reagieren sehr direkt auf Groessenordnungen. Auch Regularisierung wird unfair, wenn manche Merkmale riesige Skalen haben.
Ein Ausreisser wie 300 mm Schnabellaenge kann MinMax und StandardScaler deutlich mitziehen. RobustScaler bleibt gelassener.

Modul 4 | Neue Zutaten

Feature Expansion und Discretization

Analogie: Aus Mehl und Wasser wird Teig, und aus exakten Massen werden Groessenklassen wie S, M, L.

ca. 20 Min.

Discretization

KBinsDiscretizer verwandelt kontinuierliche Zahlen in Gruppen. Das kann Rauschen reduzieren oder lineare Modelle flexibler machen.

quantile sorgt fuer etwa gleich viele Faelle pro Bin.
uniform setzt gleich breite Intervalle.
Schlechte Bin-Grenzen kosten Information. Nutze Binning gezielt, nicht automatisch.

PolynomialFeatures

Mit degree=2 entstehen Quadrate und Interaktionsterme. Damit kann ein lineares Modell gekruemmtere Zusammenhaenge indirekt lernen.

Erst skalieren, dann expandieren. Sonst koennen die Werte stark explodieren und Regularisierung oder Distanzlogik verzerren.

Modul 5 | Rezept automatisieren

Pipeline und ColumnTransformer

Analogie: Ein Kuechenteam arbeitet parallel an unterschiedlichen Zutaten, waehrend die Pipeline die Schritte fuer jede Zutat nacheinander organisiert.

ca. 30 Min.

Klickbare Architektur

Gesamt-Pipeline

Ein einziges Objekt verwaltet Vorverarbeitung und Modell gemeinsam. Dadurch bleibt der Ablauf reproduzierbar: .fit(X_train, y_train) lernt alles auf den Trainingsdaten, .predict(X_test) transformiert nur und sagt dann vorher.

Warum ColumnTransformer und Pipeline zusammen?

ColumnTransformer arbeitet parallel auf unterschiedlichen Spaltenlisten. Ideal, wenn Textspalten anders behandelt werden als Zahlen.
Pipeline fuehrt Schritte nacheinander aus. Ideal, wenn auf derselben Spalte erst imputiert und dann skaliert werden soll.
Genau diese Kapselung schuetzt vor versehentlichen fit()-Aufrufen auf Testdaten.

Code-Beispiel

from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.linear_model import LinearRegression
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import OneHotEncoder, RobustScaler

sex_pipeline = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="most_frequent")),
    ("ohe", OneHotEncoder(drop="first", sparse_output=False))
])

flipper_pipeline = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="median")),
    ("scaler", RobustScaler())
])

preprocessor = ColumnTransformer(transformers=[
    ("sex", sex_pipeline, ["sex"]),
    ("species", OneHotEncoder(drop="first", sparse_output=False), ["species"]),
    ("flipper", flipper_pipeline, ["flipper_length_mm"]),
    ("bill", RobustScaler(), ["bill_length_mm"])
])

model = Pipeline(steps=[
    ("preprocessing", preprocessor),
    ("regression", LinearRegression())
])

Die Daten-Kueche - Zusammenfassung

Wenn alle Module abgeschlossen sind, leuchten alle Sterne auf. Der rote Faden bleibt derselbe: erst Typ erkennen, dann passende Vorbereitung, dann reproduzierbar in eine Pipeline packen.

Technik Kernfrage Warnung
ImputationWie gehe ich mit Luecken um?Nie auf Testdaten fitten.
EncodingHat die Kategorie eine Reihenfolge?Ordinal nur nutzen, wenn Reihenfolge echt ist.
ScalingDominieren grosse Zahlen die Logik?Ausreisser koennen einfache Skalen verzerren.
DiscretizationHilft Gruppierung mehr als exakter Wert?Zu grobe Bins kosten Details.
ExpansionBrauche ich Interaktionen oder Kruemmung?Erst skalieren, dann erweitern.
PipelineWie halte ich den Ablauf sauber?Einzelschritte ohne Kapselung sind fehleranfaellig.
