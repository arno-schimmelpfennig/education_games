Interaktive Steuerung: Regularisierung

Modus (Ridge/Lasso/Elastic Net) Ridge nutzt L2 und schrumpft Gewichte weich. Lasso nutzt L1 und kann Gewichte exakt auf 0 setzen. Elastic Net mischt beides und hilft oft bei stark korrelierten Merkmalen. Ridge (L2)
Lambda (0 = keine Strafe, 10 = sehr stark) Was? Erhöht oder verringert die Strafe für große Gewichte. Warum? Damit ihr den Trade-off zwischen Stabilität auf neuen Daten und möglichem Informationsverlust versteht. 0.0
l1-ratio r (nur Elastic Net) r = 0 bedeutet reines Ridge-Verhalten, r = 1 bedeutet reines Lasso-Verhalten. Werte dazwischen mischen beide Effekte. 0.50
Merksatz: Ridge hält alle Features im Spiel, Lasso kann Features ausschalten. Beide reduzieren Overfitting, wenn Lambda passend gewählt ist.

Vorteile, Grenzen und Gleichungen

Formel-Leseschlüssel
  • Grundidee: J = MSE + alpha * Penalty. Ein Teil passt Daten an, ein Teil bremst große Gewichte.
  • Ridge (L2): Penalty = Summe b_j^2, schrumpft Gewichte weich Richtung 0.
  • Lasso (L1): Penalty = Summe |b_j|, kann Gewichte exakt auf 0 setzen.
  • Warum Quadrat bei L2? Große Koeffizienten werden überproportional stärker bestraft und die Optimierung bleibt glatt.
  • Ridge/L2: Schiebt Koeffizienten Richtung 0, aber selten exakt auf 0.
  • Lasso/L1: Kann Koeffizienten auf 0 setzen und wirkt als Feature-Selektion.
  • Elastic Net: Mischt L1 und L2 über r = l1-ratio.
  • Data Generation Function unbekannt: Regularisierung hilft besonders bei streuenden Daten mit unklarer Grundfunktion.
  • Harte Einschränkung: Beispiel: ∑|b_j| ≤ t oder ∑b_j² ≤ t.
  • Weiche Einschränkung: J = MSE + α · Penalty.
  • Geometrische Intuition: L2 entspricht der Ridge-Kreis-Idee (weiches Schrumpfen), L1 der Diamant-Idee (mehr Nullsetzungen).
  • Trade-off: Zu wenig Strafe = Overfitting, zu viel Strafe = Underfitting.
  • Feature Engineering zuerst: Saubere, fachlich sinnvolle Features reduzieren Rauschen, bevor L1/L2 als Bremse wirkt.
  • Praxis: Alpha/Lambda immer über Validierung oder CV tunen (Defaultwerte sind nur Startpunkte).
  • Ziel: Bessere Generalisierung auf neuen Daten, nicht perfekte Trainingswerte.
